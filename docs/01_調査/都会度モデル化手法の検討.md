# 都会度モデル化手法の検討

## 目的

本ドキュメントは、市区町村の「都会度」を定量化するためのモデル化手法を比較検討し、データ準備の実現可能性を評価するものである。

---

## 1. 手法候補の比較一覧

| 手法 | 概要 | データ準備難易度 | 実現可能性 | 推奨度 |
|------|------|------------------|------------|--------|
| **🌙 夜間光データ（VIIRS）** | 衛星から観測される夜間光強度を都会度の代理指標とする | ⭐ 容易 | ◎ 高 | ★★★★★ |
| **🏪 店舗POIデータ** | コンビニ・スーパー・飲食店の密度から算出 | ⭐⭐ やや容易 | ◎ 高 | ★★★★☆ |

## 2. 各手法の詳細評価

### 2.1 🌙 夜間光データ（VIIRS） - **最推奨**

#### 概要
衛星から観測される夜間の地球表面の光量は、経済活動の活発さや電力消費量、インフラの整備状況と極めて高い相関を持つ。都会度を「人間活動の集積」として定義する場合、最も直接的な指標となる。

#### データソース

| ソース | URL | 形式 | 解像度 | 期間 | ライセンス |
|--------|-----|------|--------|------|------------|
| **日本VIIRS夜間光データ** | yo5uke.com | GeoTIFF (.tif.gz) | ~500m (15 arc-second) | 2012-2024 | CC BY 4.0 |
| **EOG Annual VNL V2** | mines.edu/payne-lab/eog | GeoTIFF | ~500m | 2012-present | 公開 |
| **Harmonized NTL (DMSP+VIIRS)** | Figshare/Zenodo | GeoTIFF | ~1km (30 arc-second) | 1992-2024 | CC BY 4.0 |

#### メリット
- **客観性**: 人間活動の直接的な観測値
- **データ準備の容易さ**: 日本抽出済みGeoTIFFが公開済み
- **時系列分析が可能**: 1992年から2024年まで連続データ
- **行政界に依存しない**: ピクセル単位での評価が可能

#### デメリット
- 昼間の活動を直接反映しない
- 産業地帯と住宅地の区別がつきにくい
- 季節変動（日照時間）の影響を受ける

#### 実装案
```
1. 日本VIIRS夜間光データ（2024年版）をダウンロード
2. 市区町村境界ポリゴン（TopoJSON）でZonal Statistics実行
3. 各市区町村の平均光量 → 0-100のスコアに正規化
```

---

### 2.2 🏪 店舗POIデータ - **推奨**

#### 概要
コンビニエンスストア、スーパーマーケット、飲食店の分布密度は、都市機能の集積を反映する優れた代理指標。特にコンビニは「24時間の都市活動センサー」として機能する。

#### データソース

| ソース | データ内容 | 取得方法 | コスト |
|--------|-----------|---------|--------|
| **OpenStreetMap** | 全国POI | Geofabrik / Overpass API | 無料（ODbL） |
| **Humanitarian Data Exchange** | 日本POI抽出済み | GeoJSON/Shapefile | 無料 |
| **MapFan DB** | 商業施設スポット | API契約 | 有料 |
| **ロケスマWEB** | チェーン店舗位置 | データ購入 | 有料 |

#### OpenStreetMap タグ
```
# コンビニエンスストア
shop=convenience

# スーパーマーケット  
shop=supermarket

# 飲食店
amenity=restaurant | amenity=cafe | amenity=fast_food
```

#### メリット
- **機能的都会度**: 生活利便性を直接反映
- **Walk Scoreとの親和性**: 距離減衰モデルが適用可能
- **更新頻度**: OpenStreetMapは継続的に更新

#### デメリット
- OpenStreetMapの網羅性に地域差がある
- 高品質データは商用（有料）
- 店舗の規模・業態の区別が必要

#### 実装案
```
1. Overpass APIで日本全国のコンビニ・スーパー・飲食店を取得
2. 各市区町村のポリゴン内に含まれるPOI数をカウント
3. 面積で除して密度を算出 → スコア化
   (オプション: 距離減衰関数を適用したWalk Score的計算)
```

---

### 2.3 👥 人口メッシュ統計 - **推奨**

#### 概要
国土数値情報の500mメッシュ別人口データを使用。行政界に依存しない人口分布の把握が可能。

#### データソース

| ソース | URL | 形式 | 解像度 |
|--------|-----|------|--------|
| **国土数値情報** | nlftp.mlit.go.jp | Shapefile/GeoJSON | 500m/1kmメッシュ |
| **e-Stat 地図で見る統計** | e-stat.go.jp | CSV/API | 500mメッシュ |

#### メリット
- **政府統計**: 信頼性が高い
- **無料**: ダウンロード・利用無料
- **将来推計も利用可能**: 2050年までの推計人口データあり

#### デメリット
- 更新頻度（5年ごと）
- 昼間人口は反映されにくい

#### 実装案
```
1. 国土数値情報から500mメッシュ人口データをダウンロード
2. DID（人口集中地区）相当のメッシュ数を市区町村ごとに集計
3. 高密度メッシュの割合 → 都会度スコア化
```

### 2.4 🏢 事業所密度

#### 概要
経済センサスのデータを活用し、単位面積あたりの事業所数から経済活動の集積度を評価。

#### データソース
- **e-Stat経済センサス**: 市区町村別事業所数
- **RESAS API**: 地域経済分析システム

#### メリット
- 経済活動の直接的指標
- 昼間人口（就業者）を反映

#### デメリット
- データ取得に手間（API利用登録等）
- 更新頻度が低い

---

### 2.5 � 衛星写真解析（光学画像）

#### 概要
Sentinel-2やLandsatなどの光学衛星画像から、正規化建物指数（NDBI）や正規化植生指数（NDVI）を算出し、「物理的な建物密度」や「緑の少なさ」を計測する。

#### データソース
- **Sentinel-2 / Landsat-8**: AWS Open DataやGoogle Earth Engineで無料利用可能。
- **JAXA 高解像度土地利用土地被覆図 (HRLULC)**: 衛星画像をAI解析済みのデータ。

#### 指標
- **NDBI (Normalized Difference Built-up Index)**: 建物・コンクリート等の分布を強調。
- **NDVI (Normalized Difference Vegetation Index)**: 植生活性度。逆数が都会度と相関。

#### メリデメ
- **メリット**: 無料で高解像度（10m-30m）。物理的な地表面の状態を直接反映。
- **デメリット**: 雲の影響を受ける。画像処理（大気補正、バンド演算）の専門知識が必要。データ量が膨大。

#### 💡 代替案：JAXA土地利用図の活用 - **推奨**
自分で生画像を解析する代わりに、JAXAが公開している「高解像度土地利用土地被覆図」の**「建築地域（Built-up）」**カテゴリの面積比率を使用する手法。これなら画像処理の手間なく、実質的に「衛星写真から計測した都会度」を得られる。
- **入手先**: JAXA EORC

---

### 2.6 💰 地価データ

#### 概要
「地価公示」や「都道府県地価調査」の価格データを使用。資本主義的価値観における都会度（＝土地の経済価値）を直接反映する。

#### データソース
- **国土数値情報**: 地価公示データをShapefile/GeoJSONで提供（要年度指定）。

#### メリット
- **経済的価値**: 「選ばれる場所」としての都会度を反映。
- **オープンデータ**: 国土交通省から無料で入手可能。

#### デメリット
- ポイントデータであり、面的な広がり（住宅街全体など）の推計には補間が必要。

## 3. ハイブリッドモデルの提案

単一指標ではなく、複数指標を組み合わせた**3層構造モデル**を推奨する。

### 層構造

```
┌─────────────────────────────────────────────────────┐
│  Layer 3: 補正層（Correction Layer）                │
│  ・災害リスク（ハザードマップ）                      │
│  ・地価データ（経済的価値による補正）                │
└─────────────────────────────────────────────────────┘
                        ↓
┌─────────────────────────────────────────────────────┐
│  Layer 2: 機能層（Functional Layer）                │
│  ・店舗POI密度（Walk Scoreアプローチ）               │
│  ・交通アクセス（駅・バス停からの距離）              │
│  ・道路ネットワーク密度（OpenStreetMap）             │
└─────────────────────────────────────────────────────┘
                        ↓
┌─────────────────────────────────────────────────────┐
│  Layer 1: 基盤層（Base Layer）★まず実装★           │
│  ・夜間光データ（VIIRS）                            │
│  ・JAXA土地利用図（建築地域率）                     │
└─────────────────────────────────────────────────────┘
```

| 順位 | 手法 | 理由 |
|------|------|------|

## 6. 結論

**夜間光データ（VIIRS）を第一選択として推奨する。**

理由:
1. **データ準備が最も容易**: 日本抽出済みGeoTIFFが公開されている
2. **科学的根拠**: 都市活動・経済活動との高い相関が学術的に確立
3. **行政界に依存しない**: 平成の大合併の影響を受けない
4. **時系列分析が可能**: 2012年以降の変化を追跡可能

店舗POI密度は「生活者視点での都会度」として補完的に価値が高く、Phase 2での統合を推奨する。
